๐ฆ Represents a NoSQL-style database containing multiple collections.

A Database is a container for multiple Collection instances, each representing a logical group of documents. This structure does not enforce any schema-level validation at the database level but delegates it to each individual collection.

  • handler: A mutable reference to a NosqliteErrorHandler for logging validation or duplication errors.
ยงReturns
  • Ok(()) if the collection was created and added successfully.
  • Err(NosqliteError) if:
    • A collection with the same name already exists

  • NosqliteError::DocumentInvalid if the provided schema is not a valid object

๐ฆ Removes a collection from the database by its name.

This method searches for a collection by its name and, if found, removes it from the database. If no collection with the given name exists, an error is returned and logged via the provided NosqliteErrorHandler.

ยงParameters
  • name: The name of the collection to remove.
  • handler: A mutable reference to a NosqliteErrorHandler used to record error cases.
ยงReturns
  • Ok(()) if the collection was found and successfully removed.
  • Err(NosqliteError::CollectionNotFound) if the collection does not exist.
ยงErrors
  • NosqliteError::CollectionNotFound โ occurs when no collection with the specified name is found.
ยงExample
use nosqlite_rust::engine::models::Database;
use nosqlite_rust::engine::error::NosqliteErrorHandler;

let mut db = Database::new("temp/data38.nosqlite");
let mut handler = NosqliteErrorHandler::new("temp/data38.nosqlite".to_string());

This method performs a read-only search within the databaseโs collections. If a collection with the specified name exists, a reference to it is returned. Otherwise, None is returned.

ยงReturns
  • Some(&Collection) if a collection with the specified name exists.
